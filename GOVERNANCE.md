# Governance Model

## Maintainer responsibilities
- define acceptance criteria
- review contributions
- manage releases
- enforce data quality standards

## Contributor responsibilities
- follow scope and module guidelines
- ensure legal compliance for data
- keep quality and test coverage

## Decision process
Major changes require:
- discussion
- implementation with tests
- documented evaluation results

## Development priorities
1. improve intent parsing accuracy
2. improve extraction quality
3. expand high-quality connectors
4. refine taxonomy mapping
5. improve matching relevance

Note: provider reward/incentive mechanisms are handled in private backend systems.
