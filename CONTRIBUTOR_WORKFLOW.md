# Contributor Workflow

## Step 1. Select contribution type
- prompt improvement
- new data connector
- extraction enhancement
- taxonomy improvement
- dataset contribution

## Step 2. Implement changes
- follow module guidelines
- update tests
- preserve schema compatibility

## Step 3. Run validation
- schema validation
- unit tests
- benchmark checks

## Step 4. Submit for review
Include:
- change summary
- expected impact
- test and benchmark results

## Step 5. Review process
Maintainers verify:
- correctness
- consistency
- data quality
- scope compliance

## Step 6. Merge and record
- merge PR
- update changelog
- include benchmark updates
